V0.1.0 / 10-November-2014

Main Changes

  • First Beta release of STM32F1xx HAL Drivers for all STM32F1 devices.
  • This release is in line with STM32Cube Firmware specification Rev1.0 document and with STM32F4xx, STM32F2xx , STM32F3xx, STM32F0xx & STM32L0xx HAL Drivers versions V1.0.0
  • All HAL peripherals drivers are delivered, this is an API Freeze Proposal version - expect limitations
  • CodexTracker artifacts corrections
    • STM32F1_HAL #294149 [HAL_DEF]: Add some generic defines (__NOINLINE) in stm32fXxx_hal_def.h
    • STM32F1_HAL #293996 [TIM] Remove HAL_TIM_SlaveConfigSynchronization_DMA() from HAL_TIM API
    • STM32F1_HAL #293932 PREDIV1 source selection and PLL3 activation are not well handled
    • STM32F1_HAL #291886 [RCC] Add a comment in the How to use section to mention the Peripheral enable delay
    • STM32F1_HAL #290734 CORTEX SCR SLEEPONEXIT and SEVONPEND bits management [clone from F0 Tracker 290653]
    • STM32F1_HAL #290317 [FLASH] IS_OPTIONBYTE(VALUE) is not correct if all option_OB are selected
    • STM32F1_HAL #290025 [FLASH] Clean the error context
    • STM32F1_HAL #289967 NOR: Fix HAL_NOR_GetStatus() wrong delay calculation | Clone of STM32F4_HAL #289892
    • STM32F1_HAL #289953 [GPIO] Protect init from impossible AF configuration (STM32F0_HAL #282709)
    • STM32F1_HAL #289779 GPIO : interrupt mode is not reset (clone of STM32F0_HAL #280159)
    • STM32F1_HAL #289556 [GPIO] add assert on PIN in DeInit (clone of STM32F0_HAL #289554)
    • STM32F1_HAL #287915 Add "TIM_CHANNEL_ALL" as an argument for all Encoder Start/Stop process in the comment section | STM32F4_HAL #287898

Known Limitations

  • Code static analysis with CodeSonar not yet done
  • The following HAL peripheral drivers are 'Still ongoing' regardin "Macros renaming" task: 
    • DMA
    • RTC
    • WWDG / IWDG
    • SDIO
    • CAN
    • CEC 
    • USB FS Device & USB OTG 
    • FSMC
    • Ethernet

V0.0.1 / 13-October-2014

Main Changes

  • First Alpha release of STM32F1xx HAL Drivers for all STM32F1 devices.
  • This release is in line with STM32Cube Firmware specification Rev0.9 document and with STM32F4xx, STM32F2xx , STM32F3xx, STM32F0xx & STM32L0xx HAL Drivers versions V1.0.0
  • All HAL peripherals drivers are delivered and could be classify in 3 categories:
    • Fully Finalized : Development done, fully validated & review done - API Freeze (expect in case of Cube Change request)
    • Still ongoing : Development done & under validation - API Freeze with a good level of confidence
    • Not yet Started : HAl retrieved from other platforms & L1 check not yet done
  • HAL peripherals Fully Finalized :
    • GPIO
    • DMA
    • PWR
    • RCC
    • Cortex
    • ADC
    • RTC
    • WWDG
    • IWDG
    • UART
    • Irda
    • Smartcard
    • SDIO
  • CodexTracker artifacts corrections
    • STM32F1_HAL #289695 Issue with Lin mode data length (Cloned from STM32F0_HAL #289541)
    • STM32F1_HAL #289297 [SDMMC/SD] DMA size issue (clone from stm32l4_hal #289278)

Known Limitations

  • Code static analysis with CodeSonar not yet done
  • The following HAL peripheral drivers are 'Still ongoing': 
    • FLASH - almost finalized, only final updates after review
    • DBGMCU/SYSCFG - almost finalized, only final updates after review
    • DAC - Still under development & validation ongoing
    • TIM - Still under development & validation ongoing
    • RTC - almost finalized, only final updates after review
    • I2C - almost finalized, only final updates after review
    • CEC - just started
    • USB FS Device & USB OTG - just started
  • The following HAL peripheral drivers are 'Not yet started':
    • CRC
    • SPI
    • I2S
    • CAN
    • Ethernet
    • FSMC
